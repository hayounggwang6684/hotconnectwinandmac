# LAN 스크린샷 동기화 프로젝트 작업지시서

## 1) 의도 요약 (1~3줄)
Windows에서 복사한 이미지(클립보드)를 미리보기 후 `Z:\`에 JPEG로 저장하고, macOS는 `~/IncomingShots`의 신규 JPEG를 감지해 클립보드로 자동 반영하는 이중 플랫폼 MVP를 구축한다.  
핵심 목표는 **클립보드 → 파일 → 클립보드** 흐름을 안정적으로 자동화해 양 OS 간 스크린샷 전달 시간을 최소화하는 것이다.  
환경 제약은 Python 3.11, PySide6, Pillow, watchdog, pywin32, osascript를 준수한다.

## 2) 수락 기준 (테스트 가능하게 bullet)
- [ ] Windows 앱 실행 후, 클립보드에 이미지가 들어오면 미리보기 다이얼로그가 1초 내 표시된다.
- [ ] Windows 미리보기에서 Save 클릭 시 `Z:\`에 `YYYYMMDD_HHMMSS.jpg` 형식 파일이 생성된다.
- [ ] Windows에서 클립보드가 텍스트/파일 등 이미지가 아닐 경우 저장 동작이 발생하지 않고 앱이 정상 유지된다.
- [ ] macOS 앱 실행 후 `~/IncomingShots`에 신규 `.jpg/.jpeg` 생성 시 해당 이미지가 클립보드로 반영된다.
- [ ] macOS에서 Cmd+V(미리보기 가능한 앱) 시 방금 생성된 JPEG가 붙여넣어진다.
- [ ] 동일 파일 이벤트가 연속 발생해도 중복 클립보드 반영(로그 폭주 포함)이 방지된다.
- [ ] `Z:` 미연결/권한 이슈/osascript 실패 시 앱이 크래시하지 않고 오류 로그를 출력한다.

## 3) 폴더/파일 구조 제안 (tree)
```text
hotconnectwinandmac/
├─ requirements.txt
├─ README.md
├─ win_push/
│  ├─ main.py                # Windows 엔트리포인트 (UI + 클립보드 감시 + 저장)
│  ├─ clipboard_watcher.py   # 클립보드 시퀀스 감지/필터링
│  ├─ preview_dialog.py      # PySide6 미리보기/저장 확인 다이얼로그
│  └─ save_service.py        # 파일명 생성, JPEG 저장, 예외 처리
├─ mac_clip/
│  ├─ main.py                # macOS 엔트리포인트 (watchdog + 이벤트 루프)
│  ├─ watch_handler.py       # 신규 JPEG 이벤트 처리/중복 방지
│  └─ clipboard_setter.py    # osascript 호출 래퍼
├─ shared/
│  ├─ naming.py              # YYYYMMDD_HHMMSS.jpg 파일명 규칙 공용 함수
│  └─ logging_util.py        # 공용 로깅 포맷/레벨
└─ docs/
   └─ 작업지시서.md
```

## 4) 병렬 태스크 분배안

### A. Windows Specialist
**담당 범위**
- PySide6 기반 미리보기 다이얼로그 구현
- pywin32 시퀀스 기반 클립보드 이미지 감지
- Pillow 변환 후 `Z:\`로 JPEG 저장 (`YYYYMMDD_HHMMSS.jpg`)
- `Z:` 미연결/저장 실패 예외 처리

**입력 프롬프트(복붙용)**
```text
[역할] Windows Specialist
[목표] Python 3.11 환경에서 win_push MVP 구현
[필수] PySide6 + Pillow + pywin32 사용
[기능]
1) 클립보드 변경 감지(시퀀스 번호)
2) 이미지일 때만 미리보기 다이얼로그 표시
3) Save 클릭 시 Z:\ 에 YYYYMMDD_HHMMSS.jpg 로 저장
4) 이미지가 아니면 무시
5) Z: 미연결/권한 오류 시 사용자에게 명확한 로그/알림
[산출물]
- win_push/main.py 및 모듈 분리 코드
- 수동 테스트 체크리스트
```

### B. macOS Specialist
**담당 범위**
- watchdog으로 `~/IncomingShots` 신규 JPEG 감시
- osascript로 JPEG 클립보드 반영
- 중복 이벤트 방지(동일 파일 debounce/hash/mtime 비교 중 택1)
- macOS 권한 이슈 안내 로그

**입력 프롬프트(복붙용)**
```text
[역할] macOS Specialist
[목표] Python 3.11 환경에서 mac_clip MVP 구현
[필수] watchdog + Pillow + osascript 사용
[기능]
1) ~/IncomingShots 폴더 감시
2) 새 .jpg/.jpeg 파일 생성 시 클립보드 이미지로 설정
3) 동일 파일 중복 이벤트 방지
4) osascript 실패/권한 부족 시 앱 비정상 종료 없이 오류 로그
[산출물]
- mac_clip/main.py 및 모듈 분리 코드
- Cmd+V 기준 수동 검증 가이드
```

### C. QA or DevOps
**담당 범위**
- OS별 실행/검증 절차 문서화
- 에러 케이스 재현(Z: 미연결, 비이미지 클립보드, 중복 파일 이벤트, 권한 거부)
- 릴리즈 체크리스트(의존성/실행 명령/로그 포맷)

**입력 프롬프트(복붙용)**
```text
[역할] QA/DevOps
[목표] Windows/macOS MVP 기능 검증 및 배포 준비
[검증 항목]
- Windows: 클립보드 감지, 미리보기 표시, Z:\ JPEG 저장
- macOS: IncomingShots 감시, 클립보드 반영, Cmd+V 확인
- 예외: Z: 미연결, 비이미지 클립보드, 동일 파일 중복 이벤트, 권한 이슈
[산출물]
- 테스트 리포트(성공/실패/로그)
- 재현 절차와 개선 제안
- 최종 릴리즈 체크리스트
```

## 6) 로컬 테스트 절차

### Windows: Z:\ 저장 확인, 클립보드 감지 확인
1. `python -m pip install -r requirements.txt` 실행
2. `python win_push/main.py` 실행
3. 스니핑 툴(캡처 도구 등)로 이미지를 클립보드에 복사
4. 미리보기 다이얼로그 표시 확인
5. Save 클릭
6. `Z:\`에서 `YYYYMMDD_HHMMSS.jpg` 생성 확인
7. 생성 파일 열기(손상 여부 확인)
8. 텍스트를 클립보드에 복사해 이미지 미감지(무시) 동작 확인

### macOS: ~/IncomingShots 감시 확인, 클립보드 Cmd+V 확인
1. `python -m pip install -r requirements.txt` 실행
2. `python mac_clip/main.py` 실행
3. `~/IncomingShots`에 새 `test.jpg` 파일 생성/복사
4. 터미널 로그에 클립보드 반영 메시지 확인
5. 메모/메신저/문서 앱에서 Cmd+V 실행
6. 방금 넣은 이미지가 붙여넣어지는지 확인
7. 동일 파일 재생성/빠른 연속 이벤트 시 중복 처리 방지 여부 확인

## 7) 리스크/예외 처리

### 7-1) Z: 드라이브 미연결
- **리스크**: 저장 시 `FileNotFoundError` 또는 권한 에러 발생
- **대응**: 저장 전 `Path('Z:\\').exists()` 확인, 실패 시 사용자 메시지 + 재시도 가이드 출력

### 7-2) 클립보드에 이미지가 아닌 경우
- **리스크**: 잘못된 타입 처리 시 예외 또는 불필요 팝업
- **대응**: `ImageGrab.grabclipboard()` 결과 타입 검사(`PIL.Image.Image`일 때만 처리)

### 7-3) 동일 파일 중복 감지
- **리스크**: watchdog 이벤트 중복으로 클립보드/로그 반복 반영
- **대응**: 경로+타임스탬프 기반 debounce(예: 1초), 필요 시 파일 크기/mtime 병행

### 7-4) macOS 권한(Automation/Accessibility) 필요 여부
- **리스크**: `osascript` 실행은 되지만 클립보드 반영이 제한될 수 있음
- **대응**: 최초 실행 시 권한 허용 안내 문구 제공(Automation/Accessibility), 실패 로그에 권한 설정 경로 안내
